# Project Rules

## Language

- Code identifiers must be written in English.
- Markdown documentation must be written in English.
- Code comments and Git commit messages must be written in English.

## Workflow

- Keep code and documentation synchronized in every functional change.
- Update architecture documentation when modules, runtime flow, or delivery behavior change.
- Prefer focused commits with descriptive English messages.

## Frontend Delivery

- Use Vite for local development and production builds.
- Keep production asset references aligned with the configured deployment base path.
- Avoid introducing server-side assumptions unless a backend is explicitly added to the project.
